# 每日工作记录小程序设计文档

**日期:** 2026-05-22
**项目:** daily-journal

---

## 背景

需要一个简单的工作记录工具，用于：
- 记录每日完成的工作事项
- 追踪未完成的事项
- 提前记录未来需要做的事情

UI采用手账风格，温馨简洁。

---

## 需求

### 功能需求
1. 按日期记录工作事项（纯文本）
2. 事项状态：已完成 / 待完成
3. 待完成事项显示「待完成」标签
4. 点击「待完成」标签弹出确认窗口：
   - 确认已完成 → 移除标签
   - 确认未完成 → 选择延期日期
5. 只保留最近30天的记录
6. 可以切换日期查看

### 非功能需求
- 微信小程序平台
- 使用微信云开发（免费额度）
- 响应式UI，适配手机屏幕

---

## 技术方案

### 技术栈
- **框架**: 微信小程序原生
- **云服务**: 微信云开发
  - 云数据库（免费额度：2GB）
  - 云函数（可选）

### 数据结构

```javascript
// 事项项 (items 集合)
{
  _id: String,              // 自动生成
  content: String,          // 事项内容，纯文本
  status: String,           // "completed" | "pending"
  date: String,             // 日期 "YYYY-MM-DD"
  createdAt: Number,        // 创建时间戳
  userId: String            // 用户ID（openid）
}
```

### 索引
- `userId` + `date` 组合索引，用于快速查询某一天的事项

## 核心流程

### 1. 加载页面
```
1. 获取当前日期
2. 从云数据库查询当天的事项
3. 渲染事项列表
4. 同时清理超过30天的记录
```

### 2. 添加事项
```
1. 点击「添加事项」
2. 输入内容
3. 保存到云数据库（status = pending）
4. 刷新列表
```

### 3. 标记完成
```
1. 点击「待完成」标签
2. 弹出确认窗口
3. 用户选择「确认完成」
4. 更新 status = completed
5. 刷新列表
```

### 4. 延期事项
```
1. 点击「待完成」标签
2. 弹出确认窗口
3. 用户选择「未完成」
4. 弹出日期选择器
5. 选择新日期
6. 更新事项的 date 字段
7. 刷新列表
```

### 5. 切换日期
```
1. 点击前一天/后一天
2. 更新当前日期
3. 查询新日期的事项
4. 渲染列表
```

### 6. 清理过期数据
```
1. 计算30天前的日期
2. 删除所有 date < (今天 - 30天) 的记录
3. 在以下时机触发：
   - 每次加载页面时
   - 或添加新事项时
```

## 数据保留策略

**清理时机：** 每次加载主页面时

**清理逻辑：**
```javascript
const today = new Date();
const thirtyDaysAgo = new Date(today);
thirtyDaysAgo.setDate(today.getDate() - 30);
const cutoffDate = formatDate(thirtyDaysAgo); // "YYYY-MM-DD"

db.collection('items')
  .where({
    date: db.command.lt(cutoffDate)
  })
  .remove();
```

**日期限制：**
- 日期切换最多显示到今天 + 未来7天（或30天）
- 不能访问超过30天前的日期

## 错误处理

1. **网络错误**: 显示提示，提供重试按钮
2. **数据库错误**: 记录日志，提示用户
3. **权限错误**: 提示用户重新登录
